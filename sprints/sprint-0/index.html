    Estudo Inicial

    Período: 10/09/2023 a 25/09/2023

    🔰 Descrição

    Esta sprint tem como objetivo iniciar um processo de estudo e aprendizado essencial para a equipe, cobrindo tópicos importantes, como Git, parsers, e metodologias ágeis.

    📒✔️✍ Objetivos

    • Git Fundamentals: O objetivo desta issue é iniciar um processo de estudo e aprendizado dos principais comandos do Git, uma ferramenta essencial para o controle de versão de código.
    • Parser Basics: Estudar os principais conceitos relacionados a parsers, a fim de avaliar a sua importância para todo o projeto.
    • Configuração do Ambiente de Desenvolvimento: Configurar o ambiente de desenvolvimento adequado para a equipe, permitindo que todos trabalhem de forma eficaz e colaborativa nos projetos.
    • Metodologias Ágeis: Iniciar um processo de estudo e aprendizado sobre as metodologias ágeis de gerenciamento de projetos e desenvolvimento de software.

        📗📘📚 Tarefas

        • Git Fundamentals:
          • Listar os comandos básicos do Git, como init, clone, add, commit, push, pull, merge e branch.
          • Para cada comando listado, criar uma breve descrição de sua finalidade e como é usado.
          • Recomendar recursos de aprendizado, como tutoriais, documentação oficial do Git ou cursos online, para aprofundar o entendimento de cada comando.
          • Criar exemplos práticos de uso de comandos em situações do mundo real, como criar um novo repositório, colaborar em um projeto e solucionar conflitos de merge.
          • Designar responsáveis para revisar e compreender os comandos do Git e compartilhar o conhecimento com a equipe.
        • Parser Basics:
          • Procurar referências de tutorial sobre parser.
          • Implementar um exemplo de um tutorial.
          • Olhar um código parser no querido diário.
          • Documentar tudo o que foi estudado.
          • Criar um código simples mostrando o conceito de parser.
          • Disponibilizar links úteis sobre o conceito estudado.
          • Apresentação para a equipe.
        • Configuração do Ambiente de Desenvolvimento:
          • Avaliar os requisitos específicos do projeto.
          • Selecionar uma plataforma de desenvolvimento adequada.
          • Definir um conjunto de diretrizes e boas práticas de desenvolvimento.
          • Configurar ferramentas de controle de versão.
          • Instalar e configurar IDEs (Ambientes de Desenvolvimento Integrado) ou editores de código recomendados.
          • Configurar ambientes de testes locais e/ou servidores de integração contínua, se necessário.
          • Documentar o processo de configuração do ambiente de desenvolvimento para futuras referências.
        • Metodologias Ágeis:
          • Pesquisar e selecionar as principais metodologias ágeis, como Scrum, Kanban, Lean, Extreme Programming (XP), entre outras.
          • Coletar recursos de aprendizado.
          • Designar responsáveis para cada função.
          • Realizar reuniões periódicas para discutir o progresso, compartilhar conhecimento e tirar dúvidas.
        • Critérios de Conclusão:
          • Cada membro da equipe deve ter um entendimento claro dos comandos do Git listados.
          • Os recursos de aprendizado recomendados devem ser compartilhados e disponíveis para consulta.
          • Exemplos práticos de uso dos comandos do Git devem ser criados e compartilhados.
          • O ambiente de desenvolvimento está configurado de acordo com os requisitos específicos do projeto.
          • Documentação completa e disponível para a equipe.

        ✔️ Conclusão

        • Esta sprint abrange uma ampla gama de tópicos, desde Git e parsers até a configuração do ambiente de desenvolvimento e metodologias ágeis.
