    Requisitos do Projeto

    📄 Descrição

    Baseando-se na proposta do projeto Querido Diário da Open Knowledge Brasil, nosso site tem o objetivo de tornar as informações sobre processos administrativos de municípios, geralmente publicados em documentos chamados de "Diários Oficiais", que por muitas vezes são difíceis de entender, acessíveis, para que qualquer pessoa com acesso à plataforma, consiga entender e saber como o dinheiro público está sendo utilizado.

    Para tal, decidimos trabalhar com licitações. De acordo com o Portal da Transparência, "Licitação é o processo por meio do qual a Administração Pública contrata obras, serviços, compras e alienações. Em outras palavras, licitação é a forma como a Administração Pública pode comprar e vender."

    Declaração do problema, metodologia, tecnologias e atividades

    A dificuldade de acesso aos dados de licitação nos Diários Oficiais do Estado do Piauí é um desafio significativo que impacta negativamente na transparência e eficiência dos processos públicos. A dispersão dessas informações em documentos extensos e muitas vezes complexos torna a busca e a análise um trabalho árduo, desestimulando a participação de empresas e cidadãos interessados.

    Nosso projeto visa superar essa barreira por meio da criação de uma plataforma digital centralizada e de fácil acesso. Essa plataforma consolidará os dados de licitação de forma organizada e intuitiva, oferecendo recursos de pesquisa avançada, categorização eficiente e notificações automáticas. Ao simplificar o acesso, nossa solução promoverá a transparência, permitindo que empresas e cidadãos interessados encontrem rapidamente as informações relevantes para participação em processos licitatórios.

    Além disso, a plataforma integrará ferramentas de análise de dados, facilitando a interpretação de tendências, padrões e estatísticas relacionadas às licitações. Isso não apenas economizará tempo, mas também contribuirá para a tomada de decisões informadas por parte dos stakeholders envolvidos, melhorando a eficiência e a competitividade nos processos licitatórios do Estado do Piauí.

    Acreditamos que ao eliminar as barreiras de acesso aos dados de licitação, nosso projeto será um catalisador para aprimorar a participação, promover a concorrência justa e fortalecer a integridade no âmbito das contratações públicas. A transparência gerada por nossa solução contribuirá para o fortalecimento da democracia e para o desenvolvimento socioeconômico sustentável no Estado do Piauí.

    👩🏾‍💻 Funcionalidades

    Nosso projeto oferece as seguintes funcionalidades:

    • Coleta de Diários Oficiais: O sistema realiza a coleta e acesso aos Diários Oficiais de todos os municípios do Piauí de forma eficiente e automatizada.
    • Extração de Informações de Licitações: Efetua a extração precisa de dados relacionados a licitações em todos os municípios do Piauí, garantindo a integridade das informações.
    • Análise de Dados: O projeto processa os dados extraídos, organizando-os individualmente para cada município durante a análise. Além disso, gera gráficos que destacam tendências e insights valiosos.
    • Interface de Usuário Intuitiva: Inclui uma interface de usuário amigável que permite aos usuários encontrar de maneira transparente as informações das licitações realizadas pelos municípios do Piauí, tornando a pesquisa e análise mais acessíveis.

    ⚙️Ferramentas

    • No desenvolvimento deste projeto, optamos por uma combinação estratégica de tecnologias, destacando a versão Python 3.11 🐍 como a base sólida para implementação. O Python 3.11 traz consigo a mais recente inovação na linguagem, oferecendo melhorias de desempenho, novos recursos e correções.
    • O Apache Tika 🪶, na versão 2.9.1, foi escolhido para potencializar a extração de dados de documentos heterogêneos, oferecendo suporte a uma ampla variedade de formatos. Essa capacidade é particularmente valiosa em projetos que envolvem a análise de diferentes tipos de documentos.
    • Beautiful Soup 4 🍲, uma biblioteca Python de web scraping, desempenha um papel crucial na análise e extração de dados a partir de documentos HTML e XML, garantindo uma manipulação eficiente e intuitiva.
    • A utilização de Virtualenv ⚙️ permite a criação de ambientes virtuais isolados, garantindo uma gestão eficaz de dependências e ambientes de desenvolvimento. O Pytest foi escolhido para facilitar a execução de testes automatizados, promovendo a robustez e confiabilidade do código.
    • A biblioteca Requests ⚕️ é fundamental para realizar solicitações HTTP de maneira simplificada e eficiente, facilitando a comunicação com APIs e a obtenção de dados de fontes externas.
    • A interface do usuário foi construída com tecnologias modernas, incluindo HTML5 5️⃣, CSS3 3️⃣ e JavaScript 🇯🇸, proporcionando uma experiência de usuário agradável e responsiva.

      • 🎯 Objetivos

      1. Facilitar a Navegação:

      • Garantir uma interface intuitiva que permita aos usuários navegar pelo site de maneira fácil e eficiente.
      • Implementar um sistema de categorias e tags claras para organizar as licitações, facilitando a localização de oportunidades específicas.

      2. Fornecer Informações Detalhadas:

      • Apresentar informações completas sobre cada licitação, incluindo requisitos, prazos, valores estimados e documentos necessários.
      • Garantir que as especificações técnicas e jurídicas estejam claramente descritas para permitir que os usuários compreendam os detalhes essenciais.

      3. Recursos Avançados de Filtragem:

      • Desenvolver opções avançadas de filtragem para que os usuários possam refinar suas buscas com base em setores específicos, valores estimados, regiões geográficas e outros critérios relevantes.
      • Permitir a configuração de alertas personalizados para notificar os usuários sobre licitações que correspondam aos seus interesses.

      4. Acessibilidade Móvel:

      • Garantir que o portal seja totalmente responsivo, proporcionando uma experiência de usuário consistente em dispositivos móveis.
      • Otimizar a interface para dispositivos móveis, facilitando o acesso rápido às informações mesmo quando os usuários estão em trânsito.

      5. Atualizações em Tempo Real:

      • Implementar um sistema que forneça atualizações em tempo real sobre novas licitações, modificações nos prazos e outras informações relevantes.
      • Assegurar que a plataforma esteja sempre sincronizada com as fontes oficiais de dados para fornecer informações precisas e atualizadas.

      6. Segurança e Confidencialidade:

      • Adotar medidas robustas de segurança para proteger informações sensíveis e dados confidenciais.
      • Garantir conformidade com regulamentações de privacidade e segurança de dados para promover a confiança dos usuários na plataforma.

      7. Feedback do Usuário:

      • Implementar ferramentas de feedback para que os usuários possam relatar problemas, sugerir melhorias e expressar suas necessidades.
      • Utilizar dados de feedback para realizar melhorias contínuas na plataforma, garantindo que ela esteja alinhada às expectativas e demandas dos usuários.
